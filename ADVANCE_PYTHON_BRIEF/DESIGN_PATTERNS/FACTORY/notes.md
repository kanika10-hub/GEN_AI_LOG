# Factory Pattern

## Definition

The Factory Pattern creates objects without exposing object creation logic to the client.

## Purpose

Move object creation into a dedicated factory class.

## Why Use It?

Instead of:

Dog()
Cat()

We ask the factory to create objects.

## Advantages

* Loose coupling
* Easier maintenance
* Easy extension

## Interview Definition

Factory is a creational design pattern that creates objects through a common interface without exposing the instantiation logic.
