# Singleton Pattern

## Definition

The Singleton Pattern ensures that only one instance of a class exists throughout the application.

## Purpose

Provide a single shared object that can be accessed globally.

## Why Use It?

Some resources should only exist once:

* Database Connection
* Logger
* Cache Manager
* Configuration Manager

## Advantages

* Saves memory
* Centralized access
* Prevents duplicate resources

## Disadvantages

* Global state can make testing harder

## Interview Definition

Singleton is a creational design pattern that restricts object creation to a single instance and provides a global access point to it.
